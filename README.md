# 📊 ПодписачЪ

**Self-hosted система атрибуции подписчиков Telegram и MAX каналов.**
Определяет, откуда пришёл каждый подписчик — из какой рекламной кампании, с какими UTM-метками, сколько стоил.

Готового опенсорс-аналога не существовало. Теперь существует.

[![Docker](https://img.shields.io/badge/Docker-24.0+-2496ED?logo=docker)](https://docs.docker.com/get-docker/)

## ✨ Возможности

| Функция | Описание |
|---------|----------|
| 📈 **Точная атрибуция** | Telegram ~95% (по invite_link), MAX ~80% (корреляция) |
| 🔗 **Автоинвайт-ссылки** | Уникальная ссылка для каждого визита, автоудаление по TTL |
| 📊 **UTM-аналитика** | Drill-down: source → medium → campaign |
| 💰 **Стоимость подписчика** | CPF в EUR / TON / RUB / USD для ручных ссылок |
| 🤖 **Telegram + MAX боты** | Обе платформы из одного интерфейса |
| 📋 **Публичные отчёты** | Ссылка + пароль для клиентов, настраиваемая видимость |
| 🌐 **JS-скрипт** | ~3KB, ставится на лендинг за 2 минуты |
| 🔒 **Self-hosted** | Никаких третьих сторон. Ваши данные — на вашем сервере |
| 🎨 **Тёмная/светлая тема** | Nuxt UI, адаптивный дизайн |
| 🔜 **Яндекс Метрика** | OAuth + offline conversions *(в разработке)* |
| 🔜 **Google Analytics** | Measurement Protocol v2 *(в разработке)* |

## 🚀 Быстрый старт

```bash
git clone https://github.com/VKirill/podpisach.git
cd podpisach
./scripts/install.sh
```

Откройте `http://ваш-сервер:3000` и следуйте Setup Wizard:

1. **Создайте пароль** администратора
2. **Подключите бота** — вставьте токен из @BotFather
3. **Добавьте канал** — назначьте бота администратором

Готово. Аналитика начинает собираться немедленно.

## 📋 Требования к серверу

| Параметр | Минимум | Рекомендуется |
|----------|:-------:|:-------------:|
| CPU | 1 vCPU | 2 vCPU |
| RAM | 1 GB | 2 GB |
| Диск | 10 GB SSD | 20 GB SSD |
| ОС | Ubuntu 22.04+ / Debian 12+ | Ubuntu 24.04 |
| Docker | 24.0+ | latest |
| Docker Compose | v2.20+ | latest |

VPS от 300–500 ₽/мес (Timeweb, Selectel, Hetzner) достаточно для десятков каналов.

## 🔧 Конфигурация

Все настройки — через UI. Токен бота, подключение каналов, интеграции — вводятся в браузере и хранятся в базе данных.

Редактировать `.env` файлы **не нужно**.

## 🌐 JS-скрипт для лендинга

Добавьте одну строку на сайт — система автоматически создаст уникальную invite-ссылку для каждого визита и привяжет UTM-метки к подписчику.

Генератор сниппета — в разделе «Скрипт» интерфейса.

## 🏗️ Архитектура

Три Docker-контейнера:

```
┌──────────────────────────────────────────────────────────┐
│                    Docker Compose                        │
│                                                          │
│  ┌─────────────────┐   ┌─────────────────┐              │
│  │   app (Nuxt 4)  │   │  bot (grammY)   │              │
│  │  Frontend + API │──▶│  TG + MAX боты  │              │
│  │    порт 3000    │   │  порт 3001      │              │
│  └────────┬────────┘   └────────┬────────┘              │
│           │                     │                        │
│           └──────────┬──────────┘                        │
│                      ▼                                    │
│              ┌───────────────┐                            │
│              │ PostgreSQL 16 │                            │
│              └───────────────┘                            │
└──────────────────────────────────────────────────────────┘
```

- **app** — Nuxt 4 SSR + Nitro API, весь UI и REST API
- **bot** — grammY + MAX API client, link manager, correlator
- **postgres** — PostgreSQL 16, данные в Docker volume

Контейнеры общаются через внутреннюю Docker-сеть. PostgreSQL снаружи недоступен.

## 🔄 Обновление и бэкап

```bash
# Обновить до последней версии
./scripts/update.sh

# Создать бэкап базы данных
./scripts/backup.sh
```

> ⚠️ **Важно:** Никогда не используйте `docker compose down -v` — это удалит все данные.
